# KernelEye — Research Portfolio

**Architectural Optimization of Kubernetes Using Intelligent Telemetry via eBPF**

A final-year research project portfolio site for KernelEye — a sidecar-less Kubernetes system that uses eBPF-based kernel telemetry to improve observability, traffic routing, and runtime orchestration decisions in cloud-native environments.

---

## Overview

KernelEye collects kernel-level telemetry using eBPF, processes it through an intelligent decision engine, and uses real-time signals (latency, packet loss, resource utilization) to dynamically adjust traffic routing and workload placement — without the overhead of sidecar proxies or intrusive application-level instrumentation.

**Research focus areas:**
- Kernel-level telemetry collection via eBPF
- Runtime-aware autoscaling and intelligent scheduling
- Adaptive service traffic routing
- Distributed coordination and inter-node telemetry communication

## Contact Form Setup (Vercel + Supabase)

The contact form posts to `/api/contact` (an Astro server endpoint) which inserts submissions into Supabase.

**1. Create the table in Supabase:**

```sql
create table if not exists public.contact_submissions (
  id          bigint generated by default as identity primary key,
  name        text not null,
  email       text not null,
  subject     text not null,
  message     text not null,
  ip_address  text,
  user_agent  text,
  created_at  timestamptz not null default now()
);
```

**2. Set environment variables** (locally in `.env`, and in Vercel project settings):

```env
SUPABASE_URL=https://<your-project>.supabase.co
SUPABASE_SERVICE_ROLE_KEY=<your-service-role-key>
```
