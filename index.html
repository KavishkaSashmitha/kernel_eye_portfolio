Project Scope
What is KernelEye?

KernelEye investigates how kernel-level telemetry collected through eBPF can be used to support lower-overhead and more adaptive Kubernetes orchestration.

Overview

KernelEye investigates how kernel-level telemetry collected through eBPF can be used to support lower-overhead and more adaptive Kubernetes orchestration compared to traditional sidecar-heavy approaches.

Research Problem

Existing Kubernetes observability relies heavily on sidecar proxies and userspace agents that introduce latency, resource overhead, and blind spots. How can eBPF-based kernel telemetry serve as the primary intelligence layer for adaptive orchestration decisions without intrusive instrumentation?

Solution Overview

KernelEye proposes a sidecar-less orchestration model combining eBPF telemetry, adaptive traffic routing, runtime-aware scaling, and distributed node coordination.

Low-overhead telemetry

Kernel-level signals collected with minimal runtime overhead.

Adaptive routing decisions

Traffic redirection informed by live telemetry and health signals.

Runtime-aware orchestration

Scaling and scheduling driven by real performance conditions.

Cluster-wide coordination

Node-level telemetry shared to improve holistic decisions.

Research Objectives

1. Design an eBPF-based kernel telemetry collection layer for Kubernetes with minimal overhead.

2. Develop intelligent autoscaling and scheduling driven by real-time kernel metrics.

3. Implement adaptive traffic routing responding dynamically to live telemetry signals.

4. Build a distributed coordination mechanism for inter-node telemetry sharing.

5. Evaluate overall system performance vs. standard Kubernetes deployments.

6. Publish and validate findings through academic review.
